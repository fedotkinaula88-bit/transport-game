## Файлы

- `index.html` — экран игрока (открывают все участники)
- `host.html` — экран ведущего (только у тебя, показываешь через демонстрацию экрана)
- `questions.js` — весь контент игры (клиенты, засады, события)
- `config.js` — адрес бэкенда (уже заполнен, но проверь при переносе на новый скрипт)
- `Code.gs` — бэкенд для Google Apps Script
- `images/` — портреты клиентов (maria.png, kovalevy.png, olga.png, lena.png, petrovy.png)

## Шаг 1 — Настроить бэкенд (Google Apps Script)

1. Открой Google Таблицы → создай новую таблицу
2. Расширения → Apps Script
3. Удали весь код и вставь содержимое `Code.gs`
4. Нажми **Сохранить** (💾)
5. Нажми **Развернуть → Новое развёртывание**
6. Тип: **Веб-приложение**
7. Выполнять как: **Я**
8. Доступ: **Все** (Everyone)
9. Нажми **Развернуть** → скопируй URL

Если меняешь бэкенд на новый — не забудь обновить URL в `config.js` (в кавычках!).

---

## Шаг 2 — Разместить файлы

**GitHub Pages (бесплатно, просто):**
1. Создай репозиторий на GitHub
2. Загрузи всё: `index.html`, `host.html`, `questions.js`, `config.js`, папку `images/`
3. Settings → Pages → Source: main → /root
4. Ссылка для игроков: `https://ИМЯ.github.io/РЕПО/`
5. Ссылка для ведущего: `https://ИМЯ.github.io/РЕПО/host.html`

---

## Шаг 3 — Проведение встречи

1. Открой `host.html`, вставь в поле «Вход для агентов» ссылку на `index.html` — появится QR
2. Покажи QR на экране (демонстрация в Google Meet), участники сканируют, вводят имя и выбирают машину
3. Когда все зашли — жми клиента в блоке **«Сейчас»**: Мария → Ковалёвы → Ольга
4. Для каждого клиента запускай засады по порядку — по одной на этап воронки
5. После каждой засады: **«Показать результаты участникам»** (услышишь звук в зависимости от того, какой ответ выбрало большинство)
6. Между клиентами — **«Случайное событие»**, если хочешь разрядки
7. Если время останется — разверни **«Если останется время»** и запусти Лену или Петровых
8. В конце — **«Финал — подвести итоги»**

**Важно про звук:** чтобы участники встречи тоже слышали звуковые эффекты у тебя на экране, при демонстрации экрана в Google Meet включи галочку **«Транслировать звук вкладки/экрана»**. Игроки на своих телефонах звука не получают — там всё тихо специально, чтобы не отвлекать.

## Структура контента

- **Основной раунд (25-30 мин):** Мария, Ковалёвы, Ольга × 4 этапа = 12 кейсов
- **Резерв:** Лена, Петровы (по 2 кейса, реанимация/продажа)
- Финальная засада «Исчезновение» — универсальный бонусный кейс на любого клиента

Чтобы изменить или добавить контент — редактируй `questions.js`. У каждого кейса поле `stage` определяет бейдж этапа на экране игрока, поле `tier` у клиента ("main"/"reserve") — в какой блок панели ведущего он попадёт.
